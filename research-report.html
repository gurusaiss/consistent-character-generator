2. What Actually Determines Output Accuracy?

1
Attention Architecture

How identity tokens are injected into self-attention. This is the #1 factor. The verified finding: replacing target keys with reference keys disrupts spatial layout; only injecting values (not keys) preserves both identity and structure.

~35% of accuracy
2
Fine-tuning Strategy

LoRA + textual inversion combined beats either alone. DreamBooth-LoRA maximises identity but degrades prompt-following. There is a verified trade-off: high identity ↔ low editability.

~25% of accuracy
3
Visual Encoder Quality

Text embeddings provably lack the spatial expressivity to represent character appearance. You need an image-native encoder (CLIP-I, DINOv2, SigLIP) to extract identity features, not just text.

~20% of accuracy
4
Training Data Scale & Curriculum

InstantCharacter used 10M images in a 3-phase curriculum (identity → editability → style). Scale matters, but curriculum (what you teach and in what order) matters more than raw data size.

~12% of accuracy
5
Prompt Engineering / API Provider

Prompts and provider choice have marginal impact on character consistency — they affect style and scene composition, not identity. Switching from Gemini to DALL-E will not fix a character consistency problem.

4. Combining Multiple Models — What Actually Works

There are 4 fundamentally different ways to combine models. They are not equally powerful:

❌ Output-Level Ensembling

"Generate with Gemini + DALL-E + SD, pick the best one." This is the naive approach. It does NOT improve character consistency — each model independently loses identity between generations. You're just picking the least-bad result. Not recommended.

Low impact High API cost

✅ LoRA Merging / Weight Interpolation

Train separate LoRA adapters — one for identity, one for style, one for scene composition — then merge weights arithmetically or via DARE/TIES. Verified: LoRA + textual inversion beats either alone. This is the right approach for a "best of all" model.

High impact Moderate cost

✅ Mixture of LoRA Experts (MoLE)

Route each generation through different LoRA "experts" based on the prompt type (cinematic, anime, watercolor etc). Each expert is fine-tuned for its style. A router (small classifier) decides which expert to activate. This is how you get style-versatility without degrading character identity.

High impact Scalable

⚠️ Knowledge Distillation

Train a student model to reproduce the best outputs from Gemini + DALL-E + SD (teacher ensemble). Produces a single compact model. Requires massive compute and dataset curation. Not feasible for a startup, but this is what major labs (Adobe Firefly, Midjourney v7) are doing internally.

Very high impact Requires $M+ compute

Verified claim: "Combining LoRA weight updates with textual inversion tokens outperforms using either technique alone; removing LoRA causes underfitting and reduces identity extraction performance measurably."
